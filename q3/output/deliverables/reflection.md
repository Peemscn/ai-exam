# Reflection — AI Food Assistant

งานนี้ผมตั้งหลักที่ "ผลลัพธ์ตาม requirement" ก่อน แล้วค่อยเลือก stack ที่ทำให้ได้ผลนั้น เพราะเป็น dev เลยถนัดทาง code — ใช้ Playwright scrape ข้อมูล, Python ประมวลผล, Claude วิเคราะห์, แล้วออกมาเป็นรายงาน HTML ทั้งหมดร้อยเป็น pipeline เดียว: เก็บข้อมูล → ทำความสะอาด → ให้คะแนน → วิเคราะห์ → รายงาน

ที่ทำได้ดีคือเก็บข้อมูลจริงจาก 2 แหล่ง — Google Maps (เรตติ้ง/รีวิว/ราคา) กับ OSM Overpass (ประเภทอาหาร/พิกัด/เวลาเปิด) รวม 90 ร้านย่านอโศก ทำ scoring model โปร่งใส 6 หมวด 100 คะแนนที่อธิบายสูตรได้ทุกข้อ แล้วก็ตั้งใจให้รายงาน HTML มันใช้งานได้จริงไม่ใช่แค่เปิดดู — เลยใส่ทั้งกล่อง AI Q&A (ต่อ Claude API ถามให้แนะนำร้านจาก data จริงได้) และตารางค้นหา/กรองทั้ง 90 ร้าน

งานนี้ติดเยอะกว่า q1/q2 พอสมควร เพราะ scrape เว็บจริงมันไม่นิ่ง อย่างแรก Google Maps เป็น virtual list พอ scroll การ์ดเก่าจะถูก recycle ทำให้ข้อมูลรีวิว/ราคาของร้านบน ๆ หาย ผมเลยต้องเปลี่ยนมา scrape แบบเก็บสะสม (เก็บลง Map แล้ว keep เฉพาะ record ที่ข้อมูลครบ) อย่างที่สองคือยิง API ตรงจาก server ไม่ได้ โดน 406 (น่าจะ IP ของ datacenter โดนบล็อก) แก้ด้วยการยิงผ่าน browser แทนเพราะมี header/UA ของ browser จริง ระหว่างทางก็มี browser ปิดกลางคันตอน scrape หนัก ๆ ต้อง re-navigate ใหม่ ตอน parse ข้อมูลก็เพี้ยนอยู่ 2 จุด — category ดันไปจับบรรทัด rating ส่วน address ไปจับเวลาเปิด-ปิด ต้องแก้ให้ข้าม status line ก่อนแล้วค่อยเลือกบรรทัดที่เป็นที่อยู่จริง สุดท้ายเจอร้านซ้ำ "Amritsr" กับ "Amritsr Indian Restaurant" ก็ต้องปรับ dedupe ให้จับชื่อที่ซ้อนกันได้

บทเรียนของรอบนี้คือ AI ช่วยเขียน scraper/pipeline/HTML ได้เร็วมาก แต่พอเอามา scrape เว็บจริงมันเจอ edge case เพียบ (virtual list, IP block, DOM เปลี่ยน) จะเชื่อรอบเดียวไม่ได้ ต้อง verify ทุกขั้น — เช็คจำนวน ดู sample เปิดดูรายงานจริง ก่อนไปต่อ

เรื่องเครื่องมือ ผมเลือกตามผลลัพธ์ โจทย์ยกตัวอย่าง Apify/n8n/Google Sheets แต่ผมใช้ Playwright scrape, pipeline script เป็น automation, เก็บข้อมูลเป็น JSON/CSV กับ xlsx 3 sheets แทน Google Sheets เพราะโจทย์เน้นว่าวิธีไหนก็ได้ขอแค่ได้ผลตาม requirement — ซึ่งสุดท้ายก็ครบหมด ทั้งข้อมูลจริง 2 แหล่ง automation ที่รันได้จริง scoring และรายงาน HTML

ครั้งหน้าจะเผื่อเวลาให้การ scrape มากขึ้น (เว็บจริงไม่เสถียร อาจทำ retry/checkpoint ไว้) และอ่านโจทย์ให้ละเอียดตั้งแต่แรก — รอบนี้พลาด AI Q&A ไปตอนต้นเพราะอ่านไม่ครบ กว่าจะกลับมาเติมก็เกือบจบงานแล้ว
